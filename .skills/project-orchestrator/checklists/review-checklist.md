# Review Checklist

Use after implementation.

- [ ] Original objective satisfied.
- [ ] Acceptance criteria satisfied.
- [ ] Changed files match expected scope.
- [ ] No unrelated rewrites.
- [ ] No speculative features.
- [ ] Build passes or failure is explained.
- [ ] Tests pass or failure is explained.
- [ ] Typecheck/lint pass where applicable.
- [ ] User-facing flow verified.
- [ ] Error/loading/empty states considered.
- [ ] Auth/permissions safe.
- [ ] Secrets not exposed.
- [ ] Data migrations safe and reversible enough for context.
- [ ] External integrations validated or marked untested.
- [ ] Docs/memory updated if needed.
- [ ] Residual risks documented.
- [ ] Clear verdict given: accept, accept with risks, fix required, or escalate.
