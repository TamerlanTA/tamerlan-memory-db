# Scope Control Checklist

Use before delegating or accepting changes.

- [ ] Objective is one clear outcome.
- [ ] Required files/systems are named.
- [ ] Out-of-scope items are explicit.
- [ ] No unrelated redesign.
- [ ] No broad cleanup unless requested.
- [ ] No architecture replacement without decision.
- [ ] No new dependencies unless justified.
- [ ] No speculative features.
- [ ] No changes to unrelated formatting or generated files.
- [ ] Parked ideas are moved to backlog, not slipped into the batch.

Scope drift language:

```md
This is useful, but it is outside the current batch. Park it as a later task unless it is required to satisfy the current acceptance criteria.
```
