# Pre-Task Checklist

Before meaningful work:

- [ ] Read project memory or docs.
- [ ] Identify project goal.
- [ ] Identify current state.
- [ ] Identify active stage.
- [ ] Check git status or changed files when in a repo.
- [ ] Identify likely affected files/modules.
- [ ] Identify constraints and known decisions.
- [ ] Identify unresolved risks.
- [ ] State what is out of scope.
- [ ] Confirm validation commands or manual checks.
- [ ] Decide whether this is direct implementation, planning, review, or memory sync.

Stop if:
- [ ] The task risks production data without a clear plan.
- [ ] Credentials or secrets are required but unavailable.
- [ ] The requested direction conflicts with documented project decisions.
- [ ] Acceptance criteria are too ambiguous for safe execution.
