## КРИТИЧЕСКИЕ БЛОКЕРЫ (обнаружены 2026-04-25)

### R-01: Inline кнопки подтверждения профиля не появляются ⛔
**Симптом:** После показа поста бот не выводит кнопки ✅ Publish / ⏭️ Skip.
**Вероятные причины:** WF-10 обновлён локально, но не импортирован в n8n; или `reply_markup` не принимается Telegram нодой в этом формате.
**Что делать:** Импортировать WF-10 → проверить в n8n что `reply_markup` в additional fields есть → тест.

---

### R-02: Промпт WF-09 статичен — всегда генерирует пост на первую выбранную тему ⛔
**Симптом:** При запросе "создай пост про X" — бот всегда берёт первую тему из первого запуска, не учитывает текущую тему.
**Причина:** `Build Claude Prompt` в WF-09, вероятно, жёстко берёт первый топик из Topics листа без фильтрации по запросу. Input от AI Agent не передаётся как параметр темы.
**Что делать:** Проверить как WF-06 передаёт тему в WF-09 (поле `topic` в workflowInputData) → убедиться что WF-09 использует переданный topic, а не первый из таблицы.

---

### R-03: Выбор темы из Morning Briefing не работает ⛔
**Симптом:** Бот присылает список топиков, но пользователь не может выбрать тему из него — бот не сравнивает написанное с Topics таблицей и создаёт пост только из текста запроса.
**Причина:** WF-06 AI Agent не имеет логики "найти тему по имени в Topics листе". Создаёт пост по тексту как произвольной теме.
**Что делать:** Добавить в WF-09 шаг поиска топика в Topics листе по переданному тексту темы; или дать AI Agent инструмент `lookup_topic`.

---

### R-04: Бот не различает несколько постов в очереди (post_id игнорируется) ⛔
**Симптом:** В Google Sheets несколько постов со статусом draft, но бот считает что всегда один. Не читает по post_id.
**Причина:** WF-10 `Read Draft Posts` возвращает все draft, но `Build Post Preview` берёт `items[0]` — первый найденный, без учёта навигации. Нет механизма "следующий пост".
**Что делать:** Добавить пагинацию или передавать явный post_id в запрос. Либо показывать по одному с кнопкой "следующий".

---

### R-05: Бот не обновляет статус поста при подтверждении ⛔
**Симптом:** После одобрения пост не меняет статус с `draft` на `approved`/`published` в таблице.
**Причина:** WF-10 вызывает WF-11 через `REPLACE_WITH_WF11_ID` (placeholder не заменён) — весь approve флоу падает тихо. WF-11 не запускается → статус не обновляется.
**Что делать:** Заменить `REPLACE_WITH_WF11_ID` реальным ID WF-11 → переимпортировать WF-10.

---

### R-06: Бот не читает посты со статусом `approved` для публикации ⛔
**Симптом:** Посты одобрены, но не публикуются.
**Причина:** Связана с R-05 — WF-11 не запускается вообще. Также WF-11 не проверялся на реальном Buffer аккаунте.
**Что делать:** После R-05 — запустить WF-11 вручную с тестовыми данными → проверить Buffer dashboard.

---

### R-07: Вся логика approve/update/read статусов и публикации требует полного пересмотра ⛔
**Симптом:** Комплексная проблема — несогласованность между WF-05, WF-10, WF-11 в обработке post_id, статусов, callback данных.
**Что делать:** Перепроектировать флоу полностью:
1. WF-10 show: читать конкретный post_id (или первый draft) + кнопки с post_id
2. WF-05 approve: получает post_id из callback → вызывает WF-10(approve, post_id)
3. WF-10 approve: читает пост по post_id → обновляет статус → вызывает WF-11
4. WF-11: публикует → обновляет статус на `published`
5. Нигде не использовать items[0] без явного post_id

---

### R-08: Генерация изображений не персонализирована под стиль ⚠️
**Симптом:** Изображения генерируются в общем стиле, не в стиле Tamerlan'а.
**Что делать:** Разработать визуальный стиль (цвета, элементы, композиция) → добавить в system prompt для image_description в WF-09 `Build Claude Prompt`.

---

### R-09: `REPLACE_WITH_WF11_ID` в WF-10 ⛔
**Статус:** Не заменён. Публикация заблокирована.
Открыть WF-11 в n8n → ID из URL → заменить в `WF-10_Post_Approval.json` → импортировать.

---

### R-10: WF-06, WF-10 локальные изменения не импортированы в n8n ⚠️
Файлы обновлены локально. Пока не переимпортированы — изменения не активны.
